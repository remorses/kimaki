================================================================================
                              KEY INSIGHTS
================================================================================

1. discord-adapter.ts is the ONLY file that should import heavily from
   discord.js. All other files should go through abstractions.

2. Commands should NOT import REST/Routes directly - they should call helper
   functions from a centralized API layer.

3. Test files can be temporarily exempted since they're not shipped code, but
   still worth abstracting for consistency.

4. Type imports are safe but should still be abstracted behind common interfaces
   to decouple from discord.js version changes.

5. MessageFlags is used in 30+ command files - this is the highest ROI
   abstraction point.

6. discord.js imports in forum-sync and voice-handler are acceptable because
   they're feature-specific modules (not general-purpose utilities).

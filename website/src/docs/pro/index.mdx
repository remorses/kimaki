## Quick start

### With Kimaki (automatic)

If you use the Kimaki Discord bot, the provider is already configured. Just run `/login`, select **Kimaki**, and paste your API key.

### With OpenCode (standalone)

Install the provider plugin:

```bash
npm i -g @kimaki/opencode-kimaki-provider
```

Add to your `~/.config/opencode/opencode.json`:

```json
{
  "plugin": ["@kimaki/opencode-kimaki-provider"],
  "provider": {
    "kimaki": {
      "npm": "@ai-sdk/openai-compatible",
      "api": "https://openai.kimaki.dev/v1",
      "models": {
        "kimaki": {
          "name": "Kimaki",
          "tool_call": true,
          "attachment": true,
          "limit": { "context": 1000000, "output": 65536 },
          "cost": { "input": 1.40, "output": 4.40, "cache_read": 0.26 }
        }
      }
    }
  }
}
```

Then run `/connect` in OpenCode, select **Kimaki**, and paste your API key.

### Raw API (curl, Python, any SDK)

The endpoint is fully OpenAI-compatible:

```bash
curl https://openai.kimaki.dev/v1/chat/completions \
  -H "Authorization: Bearer kimaki_xxxxx" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "kimaki",
    "messages": [{"role": "user", "content": "Hello"}]
  }'
```

Or with the OpenAI Python SDK:

```python
from openai import OpenAI

client = OpenAI(
    base_url="https://openai.kimaki.dev/v1",
    api_key="kimaki_xxxxx",
)

response = client.chat.completions.create(
    model="kimaki",
    messages=[{"role": "user", "content": "Hello"}],
)
```

- **X Articles** — create and publish long-form articles on X
- **Jitter** — export and modify Jitter animations programmatically

Skills are synced from their source repositories on every Kimaki release, so you always have the latest version.
